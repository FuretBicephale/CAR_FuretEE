Class Hierarchy

  • object: The most base type
    • wtforms.form.BaseForm: Base Form Class.
      • unreachable.NewBase
        • wtforms.form.Form: Declarative Form base class.
          • wtforms.ext.csrf.form.SecureForm: Form that enables CSRF processing via subclassing hooks.
            • flask_wtf.form.Form: Flask-specific subclass of WTForms **SecureForm** class.
              • app.forms.forms.BookForm: Used to add a book in the website with its title, author and year of publication.
              • app.forms.forms.LoginForm: Used to log a user in the website with its pseudo and password.
              • app.forms.forms.RegisterForm: Used to register a new user in the website with its pseudo, password and a confirmation of the password.
    • wtforms.meta.DefaultMeta: This is the default Meta class which defines all the default values and therefore also the 'API' of the class Meta interface.
    • flask_sqlalchemy.Model: Baseclass for custom user models.
      • unreachable.Model
        • app.models.books.Book: Represents a Book in the database with its title, as primary key, author and publication year.
        • app.models.orders.Order: Represents a client order in the database with its id, as primary key, and books.
        • app.models.orders.OrderBook: Represents the association between an order and a book with the order id and the book title @author cachera - brabant
        • app.models.users.User: Represents a registered user in the database with its pseudo, as primary key, pass and orders and its admin status @author cachera - brabant
        • app.models.users.UserOrder: Represents the association between a user and an order with the user pseudo and the order id @author cachera - brabant
    • sqlalchemy.orm.query.Query: ORM-level SQL construction object.
      • flask_sqlalchemy.BaseQuery: The default query object used for models, and exposed as :attr:`~SQLAlchemy.Query`.
